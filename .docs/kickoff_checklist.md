# Project Kickoff Checklist

## Client-Provided Items

### Infrastructure & Access (Priority: Immediate)
- [ ] GitHub organization access - Add team members to client's org
- [ ] Admin Gmail account - Provide credentials or create dedicated admin email
- [ ] Infrastructure provider selection - Choose AWS, GCP, or Azure
- [ ] Billing setup on infrastructure provider - Payment method configured
- [ ] Weekly meeting schedule - Confirm recurring time slot

### Communication & Points of Contact
- [ ] Primary point of contact (client side) - Name, email, phone
- [ ] Secondary/escalation contact - For urgent issues
- [ ] Ad-hoc communication channel - Slack, Discord, Teams, etc.
- [ ] Channel/workspace access - Invites sent to team
- [ ] Response time expectations - SLA for messages
- [ ] Emergency contact protocol - After-hours/critical issues

### Domain & DNS
- [ ] Domain registrar access - Credentials or delegate access
- [ ] DNS management access - Ability to modify DNS records

### Existing Third-Party Accounts (if any)
- [ ] Existing API keys/credentials for services already in use

### Assets & Brand
- [ ] Logo files (SVG, PNG formats)
- [ ] Brand guidelines / style guide
- [ ] Color palette and typography preferences
- [ ] Existing design mockups or wireframes

### Business Context
- [ ] Content/copy for initial pages
- [ ] Competitor or reference site examples
- [ ] User personas or target audience info
- [ ] Primary point of contact for questions

### Organizational Data
- [ ] Vendor lists - Current vendors and service providers
- [ ] Key integrations - Existing systems that need to connect
- [ ] Useful interactions documentation - Workflows, processes, user journeys
- [ ] Existing data exports - Any data to migrate or reference
- [ ] Stakeholder map - Who needs visibility or sign-off

---

## Internal Action Items

### Scope of Work Discovery
- [ ] Review and understand full scope of work requirements
- [ ] Identify all features and functionality needed
- [ ] Map features to technical requirements
- [ ] Identify unknowns and questions for client clarification
- [ ] Document technical architecture decisions needed

### Third-Party Systems Research Initiative
- [ ] Identify all third-party systems required based on scope discovery
- [ ] Evaluate options for each category (auth, payments, email, etc.)
- [ ] Document selection criteria and recommendations
- [ ] Present options to client for final selection
- [ ] Set up accounts and integrations once approved

---

## Kickoff Meeting Agenda

1. Introductions and roles
2. Review project scope and timeline
3. Walk through this checklist
4. Confirm communication channels
5. Schedule first weekly sync
